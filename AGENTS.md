## OVERVIEW
Terraform + Terragrunt project that declaratively manages multiple GitHub organisations
(DivitMittal, Qezta) with the GitHub provider, plus a Nix flake for tooling and CI generation.
Terragrunt DRYs up the backend configuration and provider generation across orgs.

## TOKENS
Each org uses a distinct PAT (Personal Access Token):
- **DivitMittal**: `GITHUB_TOKEN_DIVITMITTAL` env var (falls back to `GITHUB_TOKEN`)
- **Qezta**: `GITHUB_TOKEN_QEZTA` env var (falls back to `GITHUB_TOKEN`)

Tokens must have `repo` + `admin:org` scopes on the target org.

## CONVENTIONS
- Terraform >=1.7 is required for `import` blocks.
- `locals.tf` is the canonical source for each org's repo list and branch protection rules.
- The `modules/branch-protection` module receives `protected_branches` as a variable (not hardcoded).
- `prevent_destroy = true` guards every repository resource.
- Workflows in `.github/workflows/` are generated from `flake/actions/`.
- `.pre-commit-config.yaml` is generated by `flake/checks.nix`.

## ANTI-PATTERNS (THIS PROJECT)
- Do not edit `.github/workflows/*.yml` by hand (generated).
- Do not modify `.pre-commit-config.yaml` manually (generated).
- Do not remove `prevent_destroy` from repo resources.
- Do not recreate existing repos; use `import` blocks to adopt them.
- Do not run `terraform` directly ‚Äî always use `terragrunt` from inside an org directory.
- Do not edit `orgs/*/terraform.tfstate` manually.

## COMMANDS
```bash
# Enter devshell (provides terraform, terragrunt, gh, nix tooling)
nix develop

# Per-org workflow
cd orgs/DivitMittal   # or orgs/Qezta
terragrunt init
terragrunt plan
terragrunt apply

# Run across all orgs at once
terragrunt run-all plan
terragrunt run-all apply

# Nix
nix fmt
nix flake check
nix run .#render-workflows
```

## NOTES
- State files live at `orgs/<org>/terraform.tfstate` (local backend).
- Terragrunt generates `backend.tf`, `versions.tf`, and `provider.tf` in each org directory at init/plan time. These are `.terraformignore`-d automatically.
- To switch to a remote backend (e.g. GCS), update `remote_state` in the root `terragrunt.hcl` only ‚Äî all orgs inherit it.
