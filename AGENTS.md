## OVERVIEW
Terraform project that declaratively manages the DivitMittal GitHub org (repos + branch protections) with the GitHub provider, plus a Nix flake for tooling and CI generation.

## STRUCTURE
```
./
├── main.tf                  # provider config, imports, state moves, module wiring
├── locals.tf                # canonical repos map (single source of truth)
├── modules/
│   ├── repositories/        # github_repository resources + validation
│   └── branch-protection/    # github_branch_protection rules
├── flake/                    # devshell, formatters, pre-commit, CI generation
├── .github/workflows/        # generated by flake/actions
└── terraform.tfstate         # local state (if using local backend)
```

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Add/modify repos | `locals.tf` | Edit repo map only; import handled in `main.tf`. |
| Repo resource behavior | `modules/repositories/main.tf` | Prevent-destroy, auto-init ignores, description check. |
| Branch protection scope | `modules/branch-protection/locals.tf` | Only repo + branch names. |
| Branch protection rules | `modules/branch-protection/main.tf` | Uniform rule set. |
| State migrations/imports | `main.tf` | `moved` + `import` blocks. |
| Nix devshell/formatters | `flake/devshells.nix`, `flake/formatters.nix` | Terraform + Nix tooling. |
| CI workflow sources | `flake/actions/*.nix` | Generates `.github/workflows/*.yml`. |

## CONVENTIONS
- Terraform >=1.7 is required for `import` blocks (`main.tf`).
- `locals.tf` is the canonical repo list; modules consume it via inputs.
- `moved` blocks in `main.tf` migrate state from old root addresses.
- `prevent_destroy = true` guards every repository resource.
- Workflows in `.github/workflows/` are generated from `flake/actions/`.
- `.pre-commit-config.yaml` is generated by `flake/checks.nix`.

## ANTI-PATTERNS (THIS PROJECT)
- Do not edit `.github/workflows/*.yml` by hand (generated by actions.nix).
- Do not modify `.pre-commit-config.yaml` manually (generated file).
- Do not remove `prevent_destroy` from repo resources.
- Do not recreate existing repos; use `import` blocks to adopt them.

## UNIQUE STYLES
- Repo catalog in `locals.tf` is grouped by domain headings for readability.
- Branch protection rules are uniform; only branch names vary per repo.
